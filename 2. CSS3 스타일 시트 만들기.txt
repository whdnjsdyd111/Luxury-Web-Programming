CSS 스타일 스트를 만드는 데에는 3가지 방법이 있다.

- <style></style> 태그에 스타일 시트 작성
- style 속성에 스타일 시트 작성
- 스타일 시트를 별도 파일로 작성하고, <link> 태그나 @import 로 불러서 사용

			<style> 태그로 스타일 시트 만들기

다음과 같은 예가 있다.

<head>
	<style>
		span { color: purple; }
	</style>
</head>

이 방법은 다음과 같은 특징이 있다.

- <style> 태그는 반드시 <head> 태그 내에서만 작성
- <style> 태그는 여러 번 작성 가능하며 스타일 시트들이 합쳐서 적용됨
- <style> 태그에 작성된 스타일 시트는 웹 페이지 전체에서 적용됨

			외부 스타일 시트 파일 불러오기

CSS3 스타일 시트만 떼어내서 .css 확장자를 가진 파일에 저장해놓고, 필요한 웹 페이지에서 불러 사용할 수 있다.
이것은 동일한 CSS3 스타일을 웹 페이지마다 중복 작성하는 불편함을 해소해주고 전체 웹 페이지 모양에 일관성을 준다.
그러므로 많은 웹 사이트들이 이 방법을 사용한다. 파일에 저장된 CSS3 스타일 시트를 불러오는 방법은 다음 2가지가 있다.

- <link> 태그 이용
- @import 이용

		<link> 태그 이용

<link> 태그는 다음과 같이 <head>에서만 사용되며 종료 태그가 없다.

<head>
	<link href="mystyle.css" type="text/css" rel="stylesheet">
</head>

- href는 CSS3 파일의 위치이며 웹 사이트에 있는 경우, 웹 사이트의 URL을 기입한다.
- type="text/css"은 CSS 언어로 작성된 텍스트 파일임을 알려준다.
- rel="stylesheet"는 불러오는 파일이 스타일 시트임을 알려준다.
- <link> 태그를 여러 번 사용하여 여러 CSS 파일을 불러 올 수 있다.

대부분 브라우저는 .css 파일을 CSS3 파일로 자동 인식하므로 .css로 작성하는 것이 바람직하다.

		@import 이용

@import 문은 <style> 안에서만 사용되며 여러 번 사용할 수 있다.
mystyle.css 파일을 불러오는 예는 다음과 같다.

<style>
	@import url(mystyle.css)
	@import "mystyle.css"
</style>

			CSS3 규칙

		CSS3 스타일은 부모 태그로부터 상속된다.

자신을 둘러싸는 태그를 부모 태그 혹은 부모 요소라고 한다.
예로 다음 <em> 태그는 <p> 태그의 자식이다.

<p style="color: green">안녕하세요
	<em style="font-size: 25px"> 자식입니다. </em>
</p>

이 코드에서 <em> 태그는 '자식입니다'의 글자가 25px 크기로 출력되고 부모인 <p> 태그를 상속받아 green으로 출력된다.
페이지 전체 태그에 적용하고 싶은 스타일이 있다면 <body> 태그에 작성하면 된다.

Example 4-05		부모 스타일의 상속

		스타일 합치기(cascading)와 오버라이딩(overriding)

브라우저는 각 태그에 대한 디폴트 스타일 시트를 가지고 있어, 개발자가 만든 스타일이 없으면 이를 적용한다.
그러므로 각 태그에는 다음 4가지 종류 스타일 시트가 동시에 적용될 수 있다.

- 브라우저의 디폴트 스타일
- .css 스타일 시트 파일에 작성된 스타일
- <style></style> 태그에 작성된 스타일
- style 속성에 작성된 스타일

스타일 합치기(cascading)란 앞의 4가지 스타일 시트가 태그에 동시에 적용될 때 스타일이 합쳐져서 적용됨을 말한다.
하지만 여러 스타일 시드들이 하나의 태그에 중첩되어 동일한 CSS3 프로퍼티에 서로 다른 값을 설정하는 충돌이 발생하기도 한다.

스타일 오버라이딩(overriding)은 덮어쓰기로, 동일한 CSS3 프로퍼티에 서로 다른 값을 설정하는 충돌 시 우선 순위가 높은 스타일을 적용하는
규칙을 말한다. 다음은 우선순위를 표현한 것으로 밑으로 갈 수록 순위가 높다
